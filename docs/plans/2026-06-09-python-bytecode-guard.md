# Python Bytecode Guard

status: completed

## Context

The sparse repository uses a Python static checker and already runs it with
`PYTHONDONTWRITEBYTECODE=1` through the Makefile gates. Local direct checker
runs can still leave ignored `__pycache__` output behind, which makes a
docs-only placeholder look less clean than intended.

## Objectives

- Add a sparse baseline check for leftover Python bytecode.
- Keep the existing SDK-free `make lint`, `make test`, `make build`, and
  `make check` gates unchanged.
- Document that generated Python bytecode is local tooling output.
- Remove existing generated bytecode from the workspace.

- `make test`
- `make build`
- `make check`
